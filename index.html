2016

Morphological analyzer for Russian python sklearn numpy machine learning More

The project was a part of programming course assignment. The idea was to replicate the results described in one of the articles from the Association of Computational Linguistics conferences. The goal of this project was to create a fast Part-of-Speech analyzer for Russian by using machine learning techniques as described in Moore (2014). The analyzed was developed by a team of 3 students (including me), we used Git, Python. The Part-of-Speech tagger was learnt with scikit-learn. More information and description can be found in the project wiki.

2015 - 2016

Recognition of Russian handwritten text python sklearn skimage machine learning More

Modern research in Russian OCR is focused on recognition of machine-printed or hand-printed text. Quite a few OCR systems can correctly recognize Russian cursive handwriting, however all such systems are commercial and closed for scientific community. This project aims at developing an open OCR system for Russian cursive script recognition. In the project I collected a dataset with alphabet characters and whole sentences for the task of Russian handwriting recognition. Such dataset would exterminate the necessity to collect training data for other Russian handwriting recognition systems and also would allow to compare their results. I also used this dataset in machine learning experiments on Russian cursive letter recognition. More information is available at the project page.

2014 – 2016

Corpus of Russian Student Texts» and «Russian Learner Corpus» corpus python django mysql More

The Corpus of Russian Student Texts (www.web-corpora.net/learner_corpus) contains texts written by students, native speakers of Russian, in academic environment (usually project writing samples from various courses). The Russian Learner Corpus (www.web-corpora.net/heritage_corpus) contains texts written by learners of Russian with various backgrounds (different native languages, heritage speakers or L2 learners). I develop the online platform for these two corpora. My goal is to create a complex system that would enable users to add and annotate texts, create subcorpora and to perform exact and lexico-grammatical search. The tools I use: Python, Django, MySQL.

2014

«Automatic Identification of Semantic Aureole of Meter» machine learning russian poetry R More

“Automatic identification of semantic aureole of meter” is a final project at a course “Meter in Russian Poetry”. The work is inspired by the idea that a particular poetry meter in the poet’s mind is subconsciously connected to a particular set of topics. This idea was quite popular among some Russian researchers in the XXth century. I tried to use various machine learning techniques to cluster Russian poems by topics and see whether the clusters correspond to meter clusters. Though the answer was negative, the work drew interest of philologists and I gave a talk about the project at an International Conference of Young Philologists in Tartu.

2014

«Automatic Identification of Paradigm Gaps in Corpora» natural language processing machine learning R python More

“Automatic identification of paradigm gaps in corpora” is the topic of my third year final paper. At that time, articles of Adam Albright on paradigmatic gaps fascinated me; I also became interested in the potential of machine learning for linguistic research. These two interests resulted in a project where I used decision trees, KNN classifier and a Naïve Bayes classifier to automatically decide on the basis of accessible Russian corpus data whether a particular word has a defective paradigm or not. In this project I used Python, R and the data of Russian National Corpus.

2013 – 2014

«Creation of Corpus Instruments for Yiddish» corpus python natural language processing More

Aim: create instruments for the Yiddish language, such as a transliterator, automatic alignment of Yiddish texts with corresponding audio files, comprising a corpus out of a mere collection of texts.
Who: a team of students and professors.
My part: automatic disambiguation of Yiddish. No disambiguation programs for the Yiddish language existed before this work.
Tools that I used: We already obtained a morphological parser for Yiddish, but its output left much to be desired, so I took disambiguation upon myself. I used several disambiguation strategies (rule-based, data-driven Brill algorithm, HMM and Viterbi decoding, simple bigram model). The disambiguator is implemented in Python. Later, this work became my second year final paper.

«Database as an Instrument for Lexico-Typological Research: Verbs of Sounds ad the Models of their Semantic Derivation» lexical typology python d3.js sqlite flask More

Aim: collect a database of verbs of animal sounds; analyze the linguistic categorization of animals and the metaphorical uses of their sound verbs.

My part: collecting and analyzing the linguistic data in Tatar and Indonesian languages; developing a visualizing interface for the collected database (www.web-corpora.net/zvukimu).
Tools that I used: Python, SQLite, D3.js.
